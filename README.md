# Project Harvest

> Development repository for **CAP Market**, the digital platform of **Carrier's Abundant Produce (CAP)**.

---

## Overview

Project Harvest is the internal software development project responsible for designing, building, and maintaining **CAP Market**.

CAP Market is a digital marketplace that makes healthy food simple to buy, simple to sell, and simple to manage. It serves customers, employees, and business owners through an integrated platform that supports ordering, inventory management, packaging, deliveries, and business operations.

This repository serves as the **single source of truth** for the entire project.

# Documentation

All product decisions are documented in the `/specifications` directory.

The documentation is considered the authoritative source for:

- Product requirements
- Business rules
- User experience
- Operations
- Data models
- Design principles
- Future roadmap

Code should implement the specifications—not redefine them.

---

# Development Philosophy

Project Harvest follows a **documentation-first** approach.

The expected workflow is:

1. Define the business need.
2. Write the specification.
3. Review and refine the design.
4. Implement the solution.
5. Test against the specification.
6. Update the documentation when changes are approved.

---

# AI Collaboration

This repository is designed for collaboration with both human developers and AI coding assistants.

Before implementing new functionality, AI assistants should:

1. Read the relevant documentation in `/specifications`.
2. Treat the specifications as the source of truth.
3. Ask for clarification rather than making assumptions when requirements are ambiguous.
4. Preserve consistency with existing architecture, business rules, and design principles.
